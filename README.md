# Resume (GitHub Pages)

This repository contains a simple static resume site and a LaTeX source.

What I added:

- `index.html` and `style.css` — static HTML/CSS resume
- `.github/workflows/gh-pages.yml` — GitHub Actions workflow that deploys `index.html` and `style.css` to the `gh-pages` branch on each push to `main`.
- `.nojekyll` marker is generated by the workflow to avoid Jekyll processing.

How to publish

1. Add a GitHub remote and push `main`:

```bash
git remote add origin git@github.com:<your-username>/<your-repo>.git
git push -u origin main
```

2. The workflow will run on push and publish the site to the `gh-pages` branch. After the workflow completes, go to the repository Settings → Pages and set the source to the `gh-pages` branch (if GitHub didn't auto-enable it).

Alternative: If you prefer serving directly from `main`, you can enable Pages in Settings → Pages and choose the `main` branch and `/ (root)` as the folder. No Action is required for that.

Preview locally

Open `index.html` in your browser or from PowerShell:

```powershell
start .\index.html
```
